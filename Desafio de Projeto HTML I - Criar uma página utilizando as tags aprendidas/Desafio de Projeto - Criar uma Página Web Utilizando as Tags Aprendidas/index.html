O que é HTML?

HTML é uma linguagem de marcação utilizada para montar páginas web, ele é a estrutura do site e por conta disso é importante se atentar a algumas questões em relação a como o arquivo é organizado. Então, nesse artigo veremos algumas boas práticas úteis para melhorar a estrutura de projetos em HTML.

O que são boas práticas na programação?

São ações que servem para conseguir desenvolver aplicações faceis de manter, de ler e adicionar ou tirar itens sem atrapalhar a estrutura como um todo.

Aqui estão algumas questões para se ter em mente ao criar um site em HTML:

*Não seguir boas práticas não significa que o seu site não funciona apenas que ele não faz o uso de determinados componentes da forma como determina o manual a linguagem.

Atenção com o cabeçalho

No HTML há cabeçalhos que variam entre h1 e vai até o h6, a numeração desses cabeçalhos mostra a importância de cada um, sendo que o mais importante é o h1.

Por ser o principal, é importante evitar ter mais de um h1 (titulo principal) em seu projeto, pois ele é o componente que mecanismos de pesquisa utilizam para identificar o assunto principal da sua página web.


Não pule numerações de cabeçalho

Pular a sequência de numeração de cabeçalho (colocar um h1 e em seguida um h6) pode dificultar a leitura do código.


Tag figure para imagens

Isso ajuda para caso dê um erro no site e a imagem colocada seja perdida, o componente alt irá aparecer como texto o que pode ser de dificil leitura do usuário. Adicionar as tags figure e figcaption pode ajudar a resolver esse problema.

Utilizar o figcaption também contribui para os mecanismos de pesquisa de identificarem o conteúdo da sua página.


Não use div como cabeçalho ou rodapé

Por ser um elemento importante para a estrutura do site é preciso conhecer e utilizar as tags apropriadas para cada componente.

Nesse caso para cabeçalho header e para rodape footer.
